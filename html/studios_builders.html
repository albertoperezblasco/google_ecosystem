Studios & Builders

Three builder lanes, not one product.

Vertex AI, Google AI Studio and Google AI Edge belong to the same ecosystem, but they solve different layers of the workflow: cloud production, browser prototyping, and on-device execution.

Read this page left to right

Vertex AI is the cloud platform, Google AI Studio is the fast prompt lab, and Google AI Edge is the on-device lane. The goal here is to avoid treating them as three names for the same thing.

Same ecosystem, different job to do

Decision surface
Cloud platform cloud_sync

Vertex AI

Use this when the work needs a full ML and generative AI platform: training, deployment, shared tooling, model lifecycle and production-oriented operations.

What it is

A platform to train and deploy ML models and AI applications, with Model Garden and a shared toolset across data, science and ML engineering.

Best for

Production systems, enterprise controls, team collaboration, model tuning and workflows that continue beyond a prompt playground.

Model Garden MLOps Deployment Training
Prompt lab experiment

Google AI Studio

Use this when you want to try models quickly, iterate on prompts in the browser, tweak run settings and then export code toward the Gemini API once the prototype feels right.

What it is

A fast experimentation workspace with multiple prompt interfaces, run settings and a direct “Get code” handoff into Gemini API usage.

Best for

Prompt prototyping, chat behavior exploration, quick parameter changes and early-stage interaction design before a heavier production setup.

Prompt testing Run settings Get code Gemini API
On-device lane phone_iphone

Google AI Edge

Use this when the model should run on the device itself: mobile, web, embedded hardware or microcontrollers, with lower latency, offline behavior and more local privacy.

What it is

An edge stack for deploying AI across Android, iOS, web and embedded targets, with MediaPipe for task APIs and LiteRT for optimized runtime execution.

Best for

Apps that need local execution, hardware acceleration or cross-platform model delivery instead of a cloud builder interface.

MediaPipe LiteRT On-device Microcontrollers
