# End User Assistants

This topic covers the Google assistant surfaces aimed at end users and productivity workflows rather than developer plumbing. The shape here is different from the infrastructure topics: these are the experiences where users actually interact with Gemini, NotebookLM, and Gemini-enabled productivity tools.

## Gemini Apps

Official: [Gemini Apps](https://support.google.com/gemini/answer/13275745)

Gemini Apps is the umbrella name Google uses for the end-user Gemini experience. In this topic it is helpful to treat it as the parent layer, while the actual day-to-day surfaces are the web app and the mobile app.

### What Gemini Apps covers

| Covered surface | Why it matters |
| --- | --- |
| Gemini Web App | Main browser entry point |
| Gemini Mobile App | Phone-first assistant flow |
| Related Gemini surfaces | Chrome and other user-facing extensions of the Gemini experience |

At a practical level, Google frames Gemini Apps as a place to brainstorm, plan, draft, summarize, and work with more capable assistant flows than a single one-off chatbot page.

## Gemini Web App

Official: [Gemini Web App](https://gemini.google/about/)

Gemini Web App is the main browser-based end-user Gemini surface. This is the orbit we renamed from the broader `Gemini Apps` label because the topic page already has a separate orbit for mobile.

### What the web surface includes

| Capability | Why users care |
| --- | --- |
| General-purpose assistant chat | Everyday planning, drafting, summarizing, and question answering |
| Google Search-grounded responses | Lets the assistant answer with fresher public information when needed |
| File-aware workflows | Supports uploads and long-context work over documents and code |
| Gemini-native features | Gems, Canvas, Deep Research, and other assistant-side capabilities are surfaced here |

This is the broadest "default" assistant surface in the topic: if a user is just "using Gemini on the web", this is usually what they mean.

## Gemini Mobile App

Official: [Gemini Mobile App](https://support.google.com/gemini/answer/14554984)

Gemini Mobile App is the phone and tablet surface for Gemini. Google positions it as a mobile assistant experience that can take text, voice, images, and camera input, and on Android it can also replace Google Assistant as the primary mobile assistant.

### What the mobile surface includes

| Mobile capability | What it adds |
| --- | --- |
| Text and voice chat | Makes Gemini usable as a standard assistant on phones |
| Image and camera input | Useful for on-the-go multimodal prompts |
| Screen-aware assistance | Lets Gemini reason over what is currently on screen in some flows |
| Connected app actions | Can use connected apps like Google Workspace and YouTube Music in supported scenarios |

The mobile app matters separately from the web app because it changes interaction style: it is less about a browser workspace and more about hands-free, camera, overlay, and assistant-native behavior.

## Gemini in Chrome

Official: [Gemini in Chrome](https://www.google.com/chrome/ai-innovations/)

Gemini in Chrome is the browser-native assistant layer. Rather than opening a separate assistant destination, it helps over what the user is already doing in Chrome.

### What Gemini in Chrome is for

| Use case | What the surface does |
| --- | --- |
| Ask about open tabs | Helps compare, summarize, and reason over current browsing context |
| Auto-browse tasks | In supported plans and regions, can carry out some browsing tasks for the user |
| Work with Google services | Can connect browsing help with Calendar or YouTube-style flows |

It belongs in this topic because it is an end-user assistant surface, not a developer feature. The key distinction is that it is browser-embedded rather than a standalone app.

## NotebookLM

Official: [NotebookLM](https://notebooklm.google/)

NotebookLM is Google's source-grounded research and writing assistant. Google describes it as an AI-powered research assistant designed to help users refine and organize ideas using their own source materials.

### What makes NotebookLM different

| Dimension | NotebookLM's angle |
| --- | --- |
| Grounding model | Works from the sources you provide |
| Output style | Research synthesis, study aids, briefings, and overviews |
| Trust model | Inline citations back to provided material |
| Input types | PDFs, websites, YouTube videos, audio files, Google Docs, and Google Slides |

NotebookLM is not just "another Gemini surface." It is more specialized around evidence-backed exploration of a user-defined source set.

## Gemini in Google Workspace

Official: [Gemini in Google Workspace](https://workspace.google.com/resources/ai/)

Gemini in Google Workspace is the productivity-suite integration layer. Instead of being a single assistant destination, it shows up inside the apps where work already happens.

### Main Workspace surfaces called out by Google

| Workspace surface | What Gemini typically helps with |
| --- | --- |
| Gmail | Drafting, summarizing, inbox help |
| Docs | Writing, rewriting, outlining, synthesis |
| Slides | Presentation drafting and content shaping |
| Sheets | Formula help, analysis support, tabular assistance |
| Meet | Meeting assistance and follow-up support |
| Chat | Team communication assistance |

Why it belongs separately in this topic:

- It is not a standalone assistant app.
- It is an assistant layer embedded in productivity software.
- The value comes from being close to work artifacts, not from being a separate chat destination.

## How the surfaces differ

| If the user mainly needs... | Best fit |
| --- | --- |
| A general-purpose browser assistant | Gemini Web App |
| A phone-first assistant with voice, image, and camera input | Gemini Mobile App |
| Help directly over open tabs and browsing tasks | Gemini in Chrome |
| Source-grounded research and synthesis | NotebookLM |
| Gemini directly inside work apps | Gemini in Google Workspace |

## Practical reading of the topic

The cleanest way to read this topic is:

- `Gemini Apps` is the umbrella.
- `Gemini Web App` and `Gemini Mobile App` are the core consumer entry points.
- `Gemini in Chrome` is a browser-embedded assistant layer.
- `NotebookLM` is the specialized research assistant.
- `Gemini in Google Workspace` is the embedded productivity layer across Google's work apps.
