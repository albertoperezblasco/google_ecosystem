# Agent Designer

## TL;DR

El constructor visual low-code para diseñar y probar agentes. Encaja cuando quieres modelar la arquitectura del agente visualmente antes o además del código.

## Qué es

Agent Designer funciona como el constructor visual low-code para diseñar y probar agentes dentro de `Vertex AI Agent Builder` en el bloque `Agents Workflow Orchestration`. La suite se entiende mejor como ciclo de vida: diseñar, construir, desplegar, operar y reutilizar piezas de agentes.

## Cuándo usarlo

- Cuando quieres modelar la arquitectura del agente visualmente antes o además del código.
- Cuando el modelo de interacción que buscas se parece a `Visual agent builder` más que a una simple llamada genérica.
- Cuando ya has decidido trabajar dentro de `Vertex AI Agent Builder` y necesitas su pieza más específica.
- Cuando buscas resolver una parte concreta del flujo sin adoptar una plataforma más amplia de entrada.

## Cuándo no es la mejor opción

- Si prefieres control directo y arquitectura agentic programada, encaja mejor `Agent Development Kit`.
- Si tu problema ya es de ejecución, sesiones, memoria y operación a escala, encaja mejor `Vertex AI Agent Engine`.
- Si quieres acelerar arrancando desde ejemplos y componentes reusables, encaja mejor `Agent Garden`.

## Capacidades clave

| Capacidad | Por qué importa |
| --- | --- |
| Canvas visual | Hace visibles flujos y control lógico |
| Prueba durante diseño | Permite validar la idea mientras la construyes |
| Puente a código | Ayuda a pasar de concepto a implementación |

## Flujo típico

1. Defines el problema concreto que esta superficie debe resolver dentro del flujo.
2. Configuras la capacidad, servicio o control con la entrada y opciones mínimas necesarias.
3. Ejecutas una primera pasada y revisas la salida, métricas o comportamiento.
4. Ajustas el resto del stack alrededor de esa pieza según lo que hayas aprendido.
