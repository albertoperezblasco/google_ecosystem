# Gemini Canvas

## TL;DR

The Gemini workspace for writing, coding, and turning ideas into shareable outputs in one place. It fits when the user wants a more artifact-oriented surface than plain chat.

## What It Is

Gemini Canvas is the creation workspace inside the Gemini app experience.

It sits between ordinary assistant chat and heavier builder platforms. The current product page positions it as a place to write, code, and create apps, games, infographics, and other outputs from a prompt or source material.

## When To Use It

- When the user wants to turn prompts or research into a concrete artifact.
- When writing and coding need to happen in the same end-user surface.
- When plain chat is too lightweight but a full builder platform would be too heavy.

## When It Is Not The Best Fit

- If the workflow is mostly conversational help, use `Gemini Apps`.
- If the task is builder-facing prompt-to-app generation, use `AI Studio Build Mode` or `Firebase Studio`.
- If the need is source-grounded notebook research, use `NotebookLM`.

## Key Capabilities

| Capability | Why It Matters |
| --- | --- |
| Artifact-oriented workspace | Helps users move from prompt to something editable and shareable |
| Writing and coding in one surface | Reduces the gap between ideation and output |
| Consumer-first creation flow | Useful before stepping into developer platforms |
