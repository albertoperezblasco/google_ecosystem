# Google AI Studio

## TL;DR

El laboratorio rápido de prompts y pruebas con gemini. Encaja cuando quieres validar comportamiento, prompts y handoff a código antes de ir a una plataforma más pesada.

## Qué es

Google AI Studio funciona como el laboratorio rápido de prompts y pruebas con Gemini dentro de `Studios & Builders` y, en el catálogo, se posiciona además como laboratorio de prompts. Este bloque separa claramente laboratorio rápido, plataforma cloud y carril on-device para que no se mezclen etapas del flujo.

## Cuándo usarlo

- Cuando quieres validar comportamiento, prompts y handoff a código antes de ir a una plataforma más pesada.
- Cuando el modelo de interacción que buscas se parece a `Browser prompt lab` más que a una simple llamada genérica.
- Cuando el encaje principal está en el tema `Studios & Builders` y no en otro bloque del ecosistema.
- Cuando necesitas una superficie con responsabilidades operativas claras y no solo una capability aislada.

## Cuándo no es la mejor opción

- Si necesitas un flujo productivo en Google Cloud con despliegue, lifecycle y operaciones, encaja mejor `Vertex AI`.
- Si el modelo debe vivir en el dispositivo y te importan latencia, privacidad u offline, encaja mejor `Google AI Edge`.

## Capacidades clave

| Capacidad | Por qué importa |
| --- | --- |
| Iteración rápida | Optimiza la velocidad para probar ideas |
| Workspace web | No exige levantar primero una stack local |
| Handoff a código | Conecta bien con el paso a Gemini API |

## Flujo típico

1. Seleccionas la parte del producto o suite que mejor encaja con la etapa del trabajo.
2. Configuras recursos, herramientas, permisos o servicios conectados según el caso.
3. Validas el flujo completo con datos, previews o ejecución real.
4. Operas, iteras o escalas desde la misma plataforma cuando el sistema madura.

## Comparativa rápida

| Necesidad principal | Mejor encaje |
| --- | --- |
| Quieres validar comportamiento, prompts y handoff a código antes de ir a una plataforma más pesada | Google AI Studio |
