# Vertex AI

## TL;DR

Google’s managed cloud AI platform for training, deployment, model lifecycle, and broader AI operations. It fits when you need more than a single API call and want a platform-shaped workflow in Google Cloud.

## What It Is

Vertex AI appears in more than one part of the catalog because it is both a builder lane and a programmatic platform surface.

- In `Studios & Builders`, it is the cloud platform lane contrasted with Google AI Studio and Google AI Edge.
- In `APIs & SDKs`, it appears as the broader managed platform surface around resources, endpoints, client libraries, and operational workflows.

Those two views describe the same product from different distances. Vertex AI is where experimentation turns into managed cloud systems with platform depth, resource control, and longer-lived operational patterns.

## Where It Appears

- `Studios & Builders`: as the main full cloud platform lane.
- `APIs & SDKs`: as the managed platform surface behind broader programmatic AI work.

## When To Use It

- When your AI work already lives in Google Cloud and needs platform-level operations.
- When you need deployment, lifecycle, resources, and enterprise-shaped controls rather than only direct model calls.
- When the problem is broader than prompt experimentation or a single SDK integration.
- When you want a managed cloud environment that can support production-facing AI systems.

## When It Is Not The Best Fit

- If you mainly want fast browser experimentation, start with `Google AI Studio`.
- If you only need direct application access to Gemini, `Gemini API` is the lighter entry point.
- If you want idiomatic client libraries over a direct API surface, `Google Gen AI SDK` is the better coding layer.
- If the model must run locally on device, `Google AI Edge` is the better lane.
- If your main need is a code-first agent framework rather than the broader platform, `Agent Development Kit` is the better fit.

## Key Capabilities

| Capability | Why It Matters |
| --- | --- |
| Full managed cloud platform | Supports systems that are broader than a single endpoint or prompt workflow |
| Resource and lifecycle control | Gives teams a home for deployment, operations, and longer-lived AI assets |
| Better production fit | Makes more sense once the work is moving beyond lightweight experimentation |
| Shared platform surface | Helps multiple teams work from the same operational layer |

## Typical Flow

1. Choose the Vertex AI capability or workflow that matches the system you are building.
2. Configure the relevant resources, endpoints, datasets, or services in Google Cloud.
3. Validate the behavior with real workloads and operational expectations in mind.
4. Expand into the surrounding lifecycle features as the system matures.
