# Gemini Apps

## TL;DR

The umbrella gemini experience for end users across web and mobile surfaces. It fits when you want the overall Gemini end-user experience before choosing a specific surface.

## What It Is

Gemini Apps acts as the umbrella Gemini experience for end users across web and mobile surfaces within `End User Assistants`. This topic is about end-user surfaces for chat, browsing, productivity, and source-based work rather than developer infrastructure.

## When To Use It

- When you want the overall Gemini end-user experience before choosing a specific surface.
- When the interaction model you need looks more like `Web and mobile chat` than like a generic one-size-fits-all surface.
- When the main fit is in the `End User Assistants` block rather than in another part of the ecosystem.
- When you want a more structured surface than assembling the entire workflow from loose parts yourself.

## When It Is Not The Best Fit

- If your use case is closer to `Gemini Web App`, start there instead.
- If your use case is closer to `Gemini Mobile App`, start there instead.
- If your use case is closer to `NotebookLM`, start there instead.
- If your use case is closer to `Gemini in Google Workspace`, start there instead.

## Key Capabilities

| Capability | Why It Matters |
| --- | --- |
| Umbrella experience | Groups several end-user surfaces into one family. |
| Broad entry point | Gives users a simple place to start before choosing a narrower surface. |
| Shared product identity | Keeps several user experiences tied into one assistant story. |

## Typical Flow

1. Start from the umbrella experience and decide which end-user surface fits best.
2. Use the relevant surface for chat, productivity, browsing, or source-grounded work.
3. Refine the workflow inside that surface until the interaction pattern is clear.
4. Move to a more specific product only when the current surface stops fitting.
