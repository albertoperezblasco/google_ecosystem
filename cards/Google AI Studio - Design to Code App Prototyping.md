# Google AI Studio

## TL;DR

La superficie rápida para probar y montar una app gemini-powered en navegador. Encaja cuando quieres pasar de idea a algo ejecutable con la menor fricción posible.

## Qué es

Google AI Studio funciona como la superficie rápida para probar y montar una app Gemini-powered en navegador dentro de `Design to Code App Prototyping` y, en el catálogo, se posiciona además como superficie de prototipado y construcción. La familia agrupa superficies para pasar de idea a interfaz o de prompt a app funcional con distintos grados de fidelidad y control.

## Cuándo usarlo

- Cuando quieres pasar de idea a algo ejecutable con la menor fricción posible.
- Cuando el modelo de interacción que buscas se parece a `Browser build surface` más que a una simple llamada genérica.
- Cuando el encaje principal está en el tema `Design to Code App Prototyping` y no en otro bloque del ecosistema.
- Cuando necesitas una superficie con responsabilidades operativas claras y no solo una capability aislada.

## Cuándo no es la mejor opción

- Si quieres vibe coding rápido con runtime server-side, secrets y paquetes npm sin montar un IDE completo, encaja mejor `AI Studio Build Mode`.
- Si quieres un browser IDE con previews, integración Firebase y un camino más largo hacia producción, encaja mejor `Firebase Studio`.
- Si quieres cerrar dirección de interfaz y sistema visual antes de entrar en una implementación seria, encaja mejor `Stitch`.

## Capacidades clave

| Capacidad | Por qué importa |
| --- | --- |
| Velocidad de idea a app | Reduce setup inicial |
| Pruebas con Gemini | Permite tantear una UX funcional |
| Salida runnable | No se queda solo en prompt experimentation |

## Flujo típico

1. Seleccionas la parte del producto o suite que mejor encaja con la etapa del trabajo.
2. Configuras recursos, herramientas, permisos o servicios conectados según el caso.
3. Validas el flujo completo con datos, previews o ejecución real.
4. Operas, iteras o escalas desde la misma plataforma cuando el sistema madura.
