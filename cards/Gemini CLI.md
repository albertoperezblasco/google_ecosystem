# Gemini CLI

## TL;DR

El agente de terminal orientado a tool use y automatización. Encaja cuando tu centro de control es la shell y quieres un agente que ejecute flujos desde ahí.

## Qué es

Gemini CLI funciona como el agente de terminal orientado a tool use y automatización dentro de `Dev Tools & Coding Agents` y, en el catálogo, se posiciona además como agente de terminal. Aquí lo importante no es solo el modelo, sino dónde ocurre el trabajo: editor, navegador, terminal o un flujo más delegado y autónomo.

## Cuándo usarlo

- Cuando tu centro de control es la shell y quieres un agente que ejecute flujos desde ahí.
- Cuando el modelo de interacción que buscas se parece a `Terminal agent` más que a una simple llamada genérica.
- Cuando el encaje principal está en el tema `Dev Tools & Coding Agents` y no en otro bloque del ecosistema.
- Cuando buscas resolver una parte concreta del flujo sin adoptar una plataforma más amplia de entrada.

## Cuándo no es la mejor opción

- Si quieres seguir en tu IDE y recibir ayuda inline mientras lees o escribes código, encaja mejor `Gemini Code Assist`.
- Si quieres construir apps en un entorno alojado y colaborativo desde el navegador, encaja mejor `Firebase Studio`.
- Si quieres enviar trabajo a un agente y revisarlo después en vez de conducir cada paso, encaja mejor `Jules 2.0`.
- Si quieres un carril más agent-first que un editor asistido o una CLI clásica, encaja mejor `Antigravity`.

## Capacidades clave

| Capacidad | Por qué importa |
| --- | --- |
| Superficie de terminal | Encaja con workflows de shell y repos locales |
| Tool use | Va más allá de un prompt de texto y puede integrarse con comandos |
| Extensibilidad | MCP y un modelo más agentic lo hacen útil para automatización |

## Flujo típico

1. Defines el problema concreto que esta superficie debe resolver dentro del flujo.
2. Configuras la capacidad, servicio o control con la entrada y opciones mínimas necesarias.
3. Ejecutas una primera pasada y revisas la salida, métricas o comportamiento.
4. Ajustas el resto del stack alrededor de esa pieza según lo que hayas aprendido.
